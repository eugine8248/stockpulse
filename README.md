# stockpulse

Personalised real-time stock dashboard with custom alerts.

A single-tenant local app — runs on your laptop or a home server, polls Yahoo
Finance for your watchlist, streams price ticks to the browser over WebSocket,
fires alerts when your thresholds cross, and displays a daily stock-analysis
report (charted with Recharts) generated by an offline analyst pipeline.

## Features

- **Watchlist** — add tickers from KLSE / NASDAQ / S&P 500 via debounced symbol
  search, drag to reorder, click for an intraday chart and full quote summary.
- **Real-time price ticks** — WebSocket hub at `/ws` with per-user rooms.
  Backend polls Yahoo on a single configurable interval and broadcasts deltas.
- **Custom alerts** — `price_above`, `price_below`, `pct_change_above`,
  `pct_change_below`. Triggered alerts surface as in-app toasts and (with
  permission granted) as browser notifications.
- **Daily analysis reports** *(v0.2)* — visit `/reports` to view the latest
  stock-analysis report with a top-picks bar chart, signal pills, executive
  summary, sector flow, and individual ticker breakdowns. Prev / next arrows
  navigate to previous days. Reports live as plain markdown in
  `data/stock-reports/YYYY-MM-DD.md` and are parsed server-side into
  structured JSON.

## Stack

- **Client** — React 18 · Vite · TypeScript · Tailwind 3.4 (class-based dark
  mode) · react-router 6 · Zustand 5 · TanStack Query 5 · Recharts 2 · lucide-react.
- **Server** — Express 4 · TypeScript · Prisma 5 · **SQLite** · `ws@8` for
  WebSockets · bcryptjs + jsonwebtoken (with optional `NO_AUTH=true` bypass for
  single-user local mode).
- **Layout** — npm-workspaces monorepo (`client/`, `server/`) packaged into a
  single multi-stage Docker image.
- **External data** — Yahoo Finance unofficial endpoints. Resilient to upstream
  401 / 4xx / 5xx (returns 502 on failure instead of crashing — *v0.2.1*).

## Quick start

```bash
# 1. Install workspaces + generate Prisma client
npm install --include-workspace-root --workspaces
npm run prisma:dev    # creates the SQLite DB + applies migrations

# 2. Optional — copy env defaults
cp .env.example .env

# 3. Run client + server together
npm run dev           # Express on :3000, Vite on :5173 (proxies /api and /ws)
```

Open <http://localhost:5173>. On first launch you'll be prompted to create the
single admin user (or set `NO_AUTH=true` in `.env` to skip auth entirely for
single-user local use).

## Configuration

All env vars are optional — defaults work for a localhost single-user setup.
See `.env.example` for the full list with comments. The two you're most likely
to touch:

- `NO_AUTH` — set to `true` to bypass login entirely (single-user local mode)
- `POLL_INTERVAL_MS` — Yahoo polling cadence in ms (default `5000`)

## Build

```bash
npm run build         # tsc -b client + vite build; tsc -p server/tsconfig.json
npm start             # node server/dist/index.js (serves built client static)
```

Production deploy is via the `Dockerfile` (multi-stage) or the
`docker-compose.yml` at the repo root.

## Daily analysis reports

The Reports page reads markdown files from `data/stock-reports/<YYYY-MM-DD>.md`.
The file format matches the output of the project's offline
`stock-analysis-pm` agent pipeline — H2 sections for Executive Summary, Top
Picks (markdown table), Economy & Flow Highlights, Sector Flow, Individual
Summaries (H3 per ticker), Risks, Not Recommended. Drop new files in the
directory; the server picks them up automatically.

## Project layout

```
client/        # Vite + React SPA
server/        # Express + Prisma + WebSocket hub
prisma/        # Schema + migrations (SQLite)
data/          # SQLite DB + stock-reports/*.md (committed reports only)
```
